# 自定义组件的v-model

自定义组件的v-model (opens new window)

父组件:

<template>
  <div class="customEvents">
      <inputPrice v-model="price"></inputPrice> 
  </div>
</template>

<script>
import inputPrice from "./components/inputPrice.vue";
export default {
  data() {
    return {
      price: 100
    };
  },
  components: { inputPrice },
  watch: {
    price() {
      console.log("price is changed");
    }
  }
};
</script>

子组件——inputPrice:

<template>
//单向绑定 因为在$attrs可以获取到value的值
  <input type="text" /> 
</template>

<script>
export default {};
</script>

结果是：

子组件inputPrice并没有实现数据的双向绑定，仅仅是单向数据绑定，意味着改变price变量的值可以改变input的value，但是改变value不能改变price；

# v-model实际上是语法糖

<input type="text" v-model="price" />

等价于：

<input type="text" :value="price" @input="price = $event.target.value">

# 组件之间的v-model 模拟

父组件:

<template>
  <div class="customEvents">
      // 父向子传递value值
      <inputPrice :value="price" @input="setInputVal"></inputPrice> 
  </div>
</template>

<script>
import inputPrice from "./components/inputPrice.vue";
export default {
  data() {
    return {
      price: 100
    };
  },
  components: { inputPrice },
  methods:{
      // 4.父组件接收到子组件传的值,修改price的值,父元素的price发生变化,
      // 子组件中所有的prop都会刷新,子组件就获得新的值
      setInputVal(value){
          this.price = value;
      }
  }
};
</script>

子组件:

<template>
// 1.单向数据绑定
// 通过父向子传值,input获取到props里的value值,
// (每次父级组件发生变更时，子组件中所有的prop都将会刷新为最新的值。)
  <input type="text" :value="value" 
         @input="updateVal($event.target.value)"/> 
</template>

<script>
export default {
    props:["value"],
    methods:{
        // 2.子组件中手动输入触发,input事件
        updateVal(value){
            // 3.手动触发父组件的input事件并将值传给父组件
            this.$emit("input",value);//利用自定义事件向父组件传值传值
        }
    }
};
</script>
  • v-bind只能实现单向绑定
  • v-model（v-bind+触发的input事件）实现双向绑定

为完成以上的功能,vue使用model属性实现

父组件:

<template>
  <div class="customEvents">
      <inputPrice v-model="price"></inputPrice> 
  </div>
</template>

<script>
import inputPrice from "./components/inputPrice.vue";
export default {
  data() {
    return {
      price: 100
    };
  },
  components: { inputPrice },
};
</script>

子组件:

<template>
  <input type="text" :value="value" 
         @change="$emit('input', $event.target.value)"/> 
</template>

<script>
export default {
    model:{
        prop: "name",
        event: "input" // 触发的事件
    },
    props:{ // 注意:仍然要在props里声明这个值
        name: String
    }
};
</script>

# 1、.native

父组件中:

<template>
  <div class="customEvents">
      // 不会触发onFocus函数
      // 不触发事件的原因是这个事件是注册到inputPrice组件上的,
      // 而不是注册到input上的,
      // 而我们在触发focus事件时事件源是input.
      <inputPrice @focus="onFocus"></inputPrice> 
      // 会触发onFocus函数
      // 在一个组件的根元素上注册监听,
      // 所有focus事件注册到了input上,
      // 故会触发onFocus函数
      <inputPrice @focus.native="onFocus"></inputPrice> 
  </div>
</template>

<script>
import inputPrice from "./components/inputPrice.vue";
export default {
  data() {
    return {
      price: 100
    };
  },
  components: { inputPrice },
  methods: {
      onFocus() {
          alert("focus"):
      }
  }
};
</script>

# 2、 $listeners

.native仅仅只能将事件绑定到组件的根元素上，但并不是所有的事件源都在根元素上，为了解决这一问题，可以使用$listeners($listeners是一个对象)属性。

<template>
  <input
    type="text"
    v-on="$listeners"
  />
</template>

# .sync

在有些情况下，我们可能需要对一个 prop 进行“双向绑定”。不幸的是，真正的双向绑定会带来维护上的问题，因为子组件可以变更父组件，且在父组件和子组件都没有明显的变更来源。

这也是为什么我们推荐以 update:myPropName 的模式触发事件取而代之。举个例子，在一个包含 title prop 的假设的组件中，我们可以用以下方法表达对其赋新值的意图：

this.$emit('update:title', newTitle)

然后父组件可以监听那个事件并根据需要更新一个本地的数据 property。例如：

<text-document
  v-bind:title="doc.title"
  v-on:update:title="doc.title = $event"
></text-document>

为了方便起见，我们为这种模式提供一个缩写，即 .sync 修饰符：

<text-document v-bind:title.sync="doc.title"></text-document>

注意带有 .sync 修饰符的 v-bind 不能和表达式一起使用 (例如 v-bind:title.sync=”doc.title + ‘!’” 是无效的)。取而代之的是，你只能提供你想要绑定的 property 名，类似 v-model。

当我们用一个对象同时设置多个 prop 的时候，也可以将这个 .sync 修饰符和 v-bind 配合使用：

<text-document v-bind.sync="doc"></text-document>

这样会把 doc 对象中的每一个 property (如 title) 都作为一个独立的 prop 传进去，然后各自添加用于更新的 v-on 监听器。

将 v-bind.sync 用在一个字面量的对象上，例如 v-bind.sync=”{ title: doc.title }”，是无法正常工作的，因为在解析一个像这样的复杂表达式的时候，有很多边缘情况需要考虑。

# @hook的讲解

hook的讲解 (opens new window)

作用：实现对子组件生命周期的监听。对任意的组件都有效果，包括引入的第三方组件。

但从语法上来看，就是组件内的生命周期函数在执行结束后会 $emit 一个 hook + 生命周期名字 的自定义事件。

生命周期函数在执行结束的意思是：

每个钩子函数执行结束,就会返回一个 "hook:对应生命周期名字" 的自定义方法，可以用一下的方法触发这个"hook:"开头的自定义事件：

  • 通过 $on(eventName, eventHandler) 侦听一个事件
  • 通过 $once(eventName, eventHandler) 一次性侦听一个事件
  • 通过 $off(eventName, eventHandler) 停止侦听一个事件

# 递归使用组件（循环使用组件）

注意：

  1. 递归使用组件一定要有跳出循环的条件，否则会陷入死循环将会导致“max stack size exceeded”错误

  2. 易错点：组件的递归使用可能会报错——Unknown custom element

    1. 如果递归使用的组件只要一个是全局组件不会报错

    2. 如果递归使用的组件都是局部组件就会报错，因为（假设A是B的父组件）：A是B的父组件，即B依赖于A，又嵌套使用A组件，那么B又是A的父组件了，这就产生了矛盾。解决办法：使用异步注册组件：

      components: {
  TreeFolderContents: () => import('./tree-folder-contents.vue')
}
