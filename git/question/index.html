# question

# 1. 没有解决合并冲突就直接将代码push的问题

解决：

​ git merge --abort

​ git reset --merge再push代码

​ 修改冲突，再重新提交代码

# 2. warning: LF will be replaced by CRLF in

​ 原因：在windows下面提交和linux提交的换行符不一样，windows中的换行符为 CRLF ； 而在linux下的换行符为LF，在windows下直接回车就会出现这个提示,修改的办法是把 core.autocrlf 设置成false

解决办法：

git config --global core.autocrlf false

# 3. 解决git每次拉取都需要输入账号密码的问题

  1. 在要拉取的目录里空白处右击，点击"Git Bash Here"启动git命令行；

  2. 在命令行里执行"git config --global credential.helper store"，这会在gitconfig目录下创建一个文件，用于记录账号和密码；

  3. 再执行一次"git pull"，输入正确的账号密码，以后就不用再输了。
