# GeoJSON (opens new window)

实例：

{ "type": "FeatureCollection",
  "features": [
    { "type": "Feature",
      "geometry": {"type": "Point", "coordinates": [102.0, 0.5]},
      "properties": {"prop0": "value0"}
      },
    { "type": "Feature",
      "geometry": {
        "type": "LineString",
        "coordinates": [
          [102.0, 0.0], [103.0, 1.0], [104.0, 0.0], [105.0, 1.0]
          ]
        },
      "properties": {
        "prop0": "value0",
        "prop1": 0.0
        }
      },
    { "type": "Feature",
       "geometry": {
         "type": "Polygon",
         "coordinates": [
           [ [100.0, 0.0], [101.0, 0.0], [101.0, 1.0],
             [100.0, 1.0], [100.0, 0.0] ]
           ]
       },
       "properties": {
         "prop0": "value0",
         "prop1": {"this": "that"}
         }
       }
     ]
   }

GeoJSON是一种对各种地理数据结构进行编码的格式，基于Javascript对象表示法的地理空间信息数据交换格式。

type 当type时，必须有的属性
point / 点 “coordinates”，[x,y]
LineString / 线 “coordinates”，[[x1,y1],[x2,y2],....] (一条线不断即可)
Polygon / 面 “coordinates”，[[x1,y1],...,[x1,y1] ："首尾连接"
MultiPoint / 多点 “coordinates”，[[点1]，[点2],...]
MultiLineString / 多线 "coordinates"成员必须是一个线坐标数组的数组：[[线段1]，[线段2],...]
MultiPolygon / 多面 "coordinates"成员必须是一个面坐标数组的数组：[[面1]，[面2],...]
GeometryCollection / 几何集合 "geometries"，值是一个数组,这个数组中的每个元素都是一个GeoJSON几何对象
Feature / 特征对象 "geometry"，这个几何成员的值是上面定义的几何对象或者JSON的null值。
“properties"，这个属性成员的值是一个对象（任何JSON对象或者JSON的null值）
如果特征是常用的标识符，那么这个标识符应当包含名字为“id”的特征对象成员
FeatureCollection / 特征集合 "features"，值是一个数组。这个数组中的每个元素都是上面定义的特征对象。
crs / 坐标参考系统的对象
bbox / 边界框数组
